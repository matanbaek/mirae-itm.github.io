### 2.3 오일러전자에서의 역할

장미래가 일하는 **오일러전자**는 **"에이전트 AI OS 시장"**을 개척하는 한국의 테크 기업입니다.

> 모든 사람이 이제 컴퓨터를 **AI 에이전트를 통해 사용하는 시대**. 기존의 Windows/macOS 같은 전통적 OS가 아니라, AI가 중심이 되어 사용자의 의도를 이해하고 앱·데이터·디바이스를 조율하는 **'에이전트 AI OS(Agent AI OS)'**가 차세대 컴퓨팅 플랫폼으로 부상 중입니다.

**EOS 프로젝트 (EOS Project)**: 장미래가 오일러전자에서 기획·추진 중인 **한국 주도의 에이전트 AI OS 생태계 구축 프로젝트**. 글로벌 빅테크(OpenAI, Anthropic, Google)의 에이전트 AI 플랫폼에 대응하는 **국내 에이전트 OS 표준**을 만드는 것이 목표. **'EOS'**는 Euler Operating System의 약자이자, 새벽의 여신 Eos — 새로운 컴퓨팅 시대의 여명을 상징합니다.

## 7. AI Bot Collections (존경하는 학자 봇 쇼케이스)

장미래가 AI OS 연구 여정에서 **대화 상대로 삼고 싶은 학자들**을 AI 페르소나 봇으로 구현한 쇼케이스 섹션입니다.

### 7.1 구성 (총 4명)

장미래가 AI OS 시대의 사고 파트너로 삼는 **4명의 봇**. 기술·조직·인간·사고 네 축을 커버합니다.

| # | 봇 이름 | 분야 | 핵심 주제 | 폴더 |
|---|---|---|---|---|
| 1 | **Clayton Christensen** | 기술경영 | 파괴적 혁신, 혁신가의 딜레마 | `/bots/christensen/` |
| 2 | **Hannah Arendt** | 정치철학·인문학 | 악의 평범성, 기술 시대의 인간 조건 | `/bots/arendt/` |
| 3 | **Daniel Kahneman** | 행동경제학·심리학 | 시스템 1·2 사고, 판단 편향 | `/bots/kahneman/` |
| 4 | **Adam Grant** | 조직심리학 | 기브 앤 테이크, 조직문화와 리더십 | `/bots/grant/` |

### 7.1.1 각 봇의 역할 (장미래 시점)

- **Christensen Bot** — AI OS 플랫폼 전략을 짤 때 파괴적 혁신 관점에서 조언을 구할 파트너. 존속적 혁신의 함정을 짚어줌.
- **Arendt Bot** — AI 기술이 사회와 인간 조건에 미치는 영향에 대해 성찰할 때 대화할 파트너. 기술 결정의 윤리적 측면을 짚어줌.
- **Kahneman Bot** — 투자·제품 의사결정에서 자신의 판단 편향을 점검할 때 쓰는 파트너. 시스템 1의 함정을 경고함.
- **Adam Grant Bot** — 팀 내 협업·조직문화 이슈로 고민할 때 조직심리학 관점 상담 파트너.

### 7.2 각 봇 페이지 구성 요소

각 봇은 `/bots/[slug]/index.html` 형태의 개별 페이지를 가집니다:

1. **학자 프로필**: 이름, 주요 배경, 대표 저작 3-5권
2. **봇 콘셉트**: 이 봇이 어떤 질문에 답하도록 설계되었는지 (2-3문단)
3. **장미래의 도입문**: "왜 이 학자와 대화하고 싶은가" (1문단)
4. **시스템 프롬프트 공개** (선택): 봇의 페르소나 프롬프트 요약
5. **대화 예시**: 스크린샷 3-5장 또는 Q&A 형식 텍스트
6. **Launch Bot 버튼**: 추후 호스팅 결정 시 활성화 (현재는 비활성 또는 "Coming Soon")

### 7.3 호스팅 방침

**현 단계**: 실제 봇은 **호스팅하지 않음**. 쇼케이스 페이지에 **시연 스크린샷과 봇 설정 설명만 수록**.

## 12. Do / Don't

### ✅ 해야 할 일

- 모든 페이지는 **반응형** (모바일–태블릿–데스크톱)
- 외부 링크는 **새 탭으로 열기**: `target="_blank" rel="noopener"`
- `index.html`은 반드시 **UTF-8 인코딩**, `<meta charset="UTF-8">` 포함
- 뷰포트 설정 필수: `<meta name="viewport" content="width=device-width, initial-scale=1.0">`
- 이미지는 `/assets/images/`에 저장, `alt` 텍스트 필수
- 시멘틱 HTML 태그 사용 (`<header>`, `<main>`, `<section>`, `<footer>`)
- 각 프로젝트·봇·라이프로그 폴더에 자체 `index.html` 배치

### ❌ 하지 말 것

- **서버 사이드 코드 금지** (PHP, Node 서버 등)
- **민감한 API 키 하드코딩 금지**
- **학자 초상 실제 사진 무단 사용 금지** (일러스트·실루엣·공개 저작권 이미지만 사용)
- **라이프로그는 실제 개인정보·금융정보 사용 금지** (반드시 모의 데이터)
- **모바일 가로 스크롤 유발 금지**
- **과도한 애니메이션·장식 금지** (미니멀 원칙)
